# 多模型对比功能使用说明

##  功能概述

多模型对比功能允许你**同时使用所有7个AI模型**对同一个AFP蛋白序列进行分析,并直观地比较不同模型的回复质量、速度和准确性。

## ✨ 核心特性

- **一键对比**: 点击"多模型对比"按钮即可同时调用所有模型
- **并行执行**: 所有模型同时运行,大幅节省时间
- **可视化展示**: 以卡片形式展示每个模型的回复
- **性能指标**: 显示每个模型的响应时间(延迟)
- **错误处理**: 清晰标识失败的模型及错误原因

## 🚀 使用方法

### 1. 启动服务

```bash
# 后端服务 (端口 8000)
cd /Users/guoqunfei/Applications/PyCharm/Projects/EvoAgent-AFP/backend
python3.9 -m pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000

# 前端服务 (端口 5175)
cd /Users/guoqunfei/Applications/PyCharm/Projects/EvoAgent-AFP/frontend
npm run dev
```

### 2. 访问应用

打开浏览器访问: `http://localhost:5175/`

### 3. 使用多模型对比

#### 步骤1: 输入AFP序列
在顶部的"AFP Sequence"输入框中输入或粘贴你的AFP蛋白序列,例如:
```
DTASDAAAAAALTAANAAAAAKLTADNAAAAAAATAR
```

#### 步骤2: 输入分析目标
在聊天输入框中输入你想要分析的问题,例如:
- "分析这个序列的冰结合基序"
- "推荐提升TH活性的突变位点"
- "评估这个序列的抗冻活性潜力"
- "设计5个优化版本并对比效果"

#### 步骤3: 点击"多模型对比"按钮
点击输入框右侧的紫色 **"🧪 多模型对比"** 按钮

#### 步骤4: 等待结果
系统会同时调用以下7个模型进行分析:
1. Kimi (kimi-k2.6)
2. DeepSeek (deepseek-v4-pro)
3. GPT-5.5 (gpt-5.5)
4. Qwen3.7-Max (qwen3.7-max)
5. GLM-5.1 (glm-5.1)
6. MiniMax-M3 (MiniMax-M3)
7. GLM-5.2 (glm-5.2)

#### 步骤5: 查看对比结果
结果会以卡片网格的形式展示,每张卡片包含:
- **模型名称**: 如 "Kimi (kimi-k2.6)"
- **响应时间**: 如 "⏱ 1250ms"
- **状态标识**: ✅ 成功 / ❌ 失败
- **完整回复**: Markdown格式的AI回复内容

### 4. 关闭对比视图
点击右上角的 "✕ 关闭" 按钮可以隐藏对比结果

## 📊 API端点

### POST `/api/v1/chat/compare-models`

**请求体:**
```json
{
  "message": "分析这个AFP序列的冰结合特性",
  "model_keys": ["kimi", "deepseek", "gpt55"]  // 可选,不指定则使用所有模型
```

```json

      "error_message": null,
      "latency_ms": 1250
    },
    ...
  ],
  "total_models": 7,
  "successful": 7,
  "failed": 0
}
```

## ⚠️ 注意事项

1. **网络要求**: 需要稳定的网络连接访问API服务
2. **API配额**: 确保你有足够的API调用配额(一次对比消耗7次调用)
3. **响应时间**: 取决于最慢的模型,通常需要5-15秒
4. **并发限制**: 建议不要频繁连续调用,避免触发限流
5. **错误处理**: 如果某个模型失败,其他模型的结果仍会正常显示

## 🔧 技术实现

### 后端架构
- **Service层**: `ChatService.compare_models()` 方法负责调度所有模型
- **API层**: `/api/v1/chat/compare-models` 端点接收请求
- **Provider模式**: 通过 `build_model_provider()` 动态创建各模型的提供者实例

### 前端架构
- **Vue组件**: `DesignLab.vue` 中的 `runMultiModelComparison()` 函数
- **API封装**: `useApi.ts` 中的 `compareModels()` 方法
- **响应式状态**: 使用 Vue ref 管理对比结果和加载状态
- **Markdown渲染**: 复用现有的 `renderMD()` 函数展示富文本
