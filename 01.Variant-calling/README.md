# Variant (SV) calling pipeline

This pipeline was made for SV calling pipeline from Cyclone reads to structural variation VCF files. It takes all FASTQ files from a sample as input and it outputs SV VCFs from [Sniffles], [cuteSV], and [delly].

## System Requirements

- sniffles=2.2
- cuteSV=v2.1.2
- truvari=v5.3.0
- chopper=0.10.0
- seqkit=2.10.1
- parallel=20250622
- minimap2=2.30-r1287
- bcftools=1.22
- mingz=hewm2008[v1.12]
- samtools=1.22.1
- delly=1.5.0

## Instructions for use

### Input

This pipeline requires all FASTQ files from one sample as input, as it is optimised for CycloneSEQ long reads.

Additionally, you need to provide a reference genome to have your variants called against. The reference can't be gzipped.

### Output

The main outputs of the pipeline are:
- sniffles2.auto0.25.filter.vcf : the final VCF file for this sample.
- sniffles2.auto0.25.snf : the file used for next step to merge all samples SVs.
- cuteSV.filter.vcf : the VCF file generated by cuteSV.
- delly.filter.vcf : the VCF file generated by delly.
- sniffles2.5X.filter.vcf : the VCF file generated by sniffles2 based on 5X CycloneSEQ long reads.
- sniffles2.10X.filter.vcf : the VCF file generated by sniffles2 based on 10X CycloneSEQ long reads.
- sniffles2.15X.filter.vcf : the VCF file generated by sniffles2 based on 15X CycloneSEQ long reads.
- sniffles2.20X.filter.vcf : the VCF file generated by sniffles2 based on 20X CycloneSEQ long reads.
- sniffles2.auto0.1.filter.vcf : the VCF file generated by sniffles2 based on auto 0.1 parameters.
- sniffles2.selected.filter.vcf : the VCF file generated by sniffles2 based on customize depth based on data volume.

### How to run
First, generate a file `Path.list` with the following structure: the first column is the sample name, and the second column is the absolute path to the Cyclone FASTQ data.
Then, under the directory `01.Variant-calling`:
```
$ sh SV.detection.sh --sample <SampleName>
```
