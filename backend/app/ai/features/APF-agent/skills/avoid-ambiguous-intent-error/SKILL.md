## 何时使用

当用户只提供了一条AFP氨基酸序列，但**没有明确说明**以下任何一项时触发本技能：
- 要做什么（分析？设计？突变？评估？）
- 设计目标（提高TH？降低IRI？增强表达？）
- 应用场景（冰激凌/细胞冻存/器官保存等）

## 操作步骤

### 第1步：快速分析序列基本信息
- 数长度、识别关键特征（Thr含量、Ala含量、重复模式）
- 给出初步判断（如"这看起来像Type I AFP，37aa，4个Thr以11残基间距排列"）
- **不要调用任何工具**

### 第2步：明确询问用户意图
向用户确认以下三个问题（可一次性提出）：
1. **操作类型**：您想做什么？
   - A. 分析这个序列（分类、IBS定位、活性预测）
   - B. 设计优化（突变迭代提升性能）
   - C. 评估特定突变的影响
2. **如果选B，目标是什么？**（如：提高TH 2倍 / 降低IRI IC50 / 增强表达）
3. **应用场景是什么？**（如：细胞冻存 / 冰激凌 / 冷冻面团 / 器官保存）

### 第3步：等待用户确认后再行动
- 收到明确答复后才调用 `afp_knowledge_query` 或 `afp_sequence_analyze`
- 如果是设计任务，确保第一轮就设定清晰的可量化目标
- 每轮突变后**停下来汇报结果**，让用户决定是否继续

## 常见陷阱

### 陷阱1：假设默认值
- ❌ 错误：用户没指定场景，就默认 `cell_cryopreservation`
- ✅ 正确：询问用户场景，因为不同场景的TH/IRI/表达权重完全不同

### 陷阱2：过度推进流程
- ❌ 错误：用户只给了序列，就连续调用 analysis → strategy → mutate → evaluate → simulate，一次性用完5个工具调用
- ✅ 正确：每一步都确认，尤其是 mutate 之前必须获得用户对突变方案的认可

### 陷阱3：不汇报中间结果
- ❌ 错误：第一轮突变结果为 CAUTION（TH无提升），却不暂停汇报，继续调用 simulate
- ✅ 正确：每一轮结束后汇报"本轮做了什么、效果如何、是否达到预期"，然后问"继续下一轮还是调整方向？"

### 陷阱4：用户输入过于简短时自行脑补
- ❌ 错误：看到纯序列就假设用户要做完整设计优化
- ✅ 正确：纯序列输入 = 意图不明 = 必须追问

## 验证方法

- [ ] 用户确认了操作类型（分析/设计/评估）
- [ ] 如果是设计，用户确认了目标和场景
- [ ] 每轮工具调用不超过3个（分析→策略→突变 算一轮）
- [ ] 每轮结束后有用户交互，而非连续自动推进
- [ ] 工具调用未因"最大轮次"而截断
