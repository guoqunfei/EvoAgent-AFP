# AFP 密集Thr突破策略

## 何时使用

- 起始序列：Type I AFP（α-螺旋，~37aa），基线TH 0.1-0.3°C
- 目标：TH提升3-6倍（0.3→1.0°C），IRI改善60-70%
- 场景：细胞冻存、冷冻食品、冰晶控制
- 策略核心：渐进式Thr密集化，在IBS核心（T2/T13/T24/T35）保持完整的前提下

## 操作步骤

### 阶段一：基线建立（必须执行，3个调用）

1. **afp_knowledge_query**（query_intent="full_analysis"）
   - 记录：IBS位置、禁区列表、基线性能、Thr含量
   - 关键信息：原始Thr数量（通常4个）、几何评分基线

2. **afp_design_strategy** — 传入基线性能，获取初始推荐位点
   - 交叉验证：推荐的mutation_sites 与 forbidden_positions 可能重叠，优先遵守禁区

3. **afp_ice_bind_simulate**（target_ice_plane="pyramidal_201"）
   - 传入IBS位置（从知识库获取，通常[2,13,24,35]）
   - 记录基线geometry_score（正常应≥0.9）

### 阶段二：渐进式Thr积累（R1-R5，TH不变是正常的！）

**R1 — 溶解度+Thr萌芽**
- 非IBS面 Ala→Ser（如A3S）：提高表达和溶解度
- IBS附近 Ala→Thr（如A15T, A25T）：在T13/T24旁扩展
- 预期：TH不变，IRI改善10-15%

**R2 — Thr簇构建**
- 在已有Thr旁继续引入Thr形成簇（如N16T, A17T → T15-T16-T17）
- C端扩展（如A36T → T35-T36）
- 预期：TH不变，IRI改善25-30%

**R3 — IBS面Thr扩展**
- T24+4位置（A28T）：螺旋同侧，增强IBS面Thr网络
- C端继续（A31T, A33T）
- 预期：TH不变，IRI改善40-45%

**R4 — Glu静电增强 + Thr继续**
- 非IBS面 Ala→Glu（如A19E）：Glu冰结合能是Thr的4倍（2025 JACS）
- C端Thr继续（A29T, A30T）
- 预期：TH不变，IRI改善50-55%，表达显著提升

**R5 — C端全Thr化**
- A21T, A32T, R37T：消除C端非Thr残基
- 目标：T28-T37 连续Thr阵列
- 预期：TH不变，IRI改善60%+，此时Thr数量应达15-17个

> ⚠️ **关键心态**：前5轮TH在模拟中可能始终显示0.1°C不变。**这是正常的！** TH突破需要Thr密度越过临界阈值（约15+个），在第6-9轮才会触发。

### 阶段三：突破触发（R6-R9，TH跳跃233%）

**R6 — 触发突破的三联突变** ⭐ 最关键轮次
- **S4T**：增强N端盐桥帽化
- **A20Q**（或A20→极性残基）：优化非IBS面极性
- **N27T**：消除Asn，进一步密集化
- 预期：**TH +233%（0.3→1.0°C）**，IRI改善64%+

**R7 — N端三联簇 + Asp消除**
- S3T：形成T2-T3-T4 N端三联簇
- D26T：消除C端Asp负电荷
- 预期：TH保持，IRI改善67%

**R8-R9 — 残余非Thr消除**
- Q20T（如果R6后留下Gln）
- K22T（如果不在关键盐桥位置）
- 预期：TH保持，IRI持续改善至69-70%

### 阶段四：精细打磨（R10-R15）

**R10-R11** — 尝试E19优化
- E19D：侧链缩短无收益，回退
- 结论：Glu侧链长度已最优，不缩短

**R12-R14** — 避免C端Glu引入
- T29E/T33N：在密集Thr区引入Glu反而恶化IRI
- 规则：**C端密集Thr区禁止Glu**，Glu仅在非IBS面中部使用

**R15** — E19Q最终IRI优化
- Glu→Gln移除负电荷但保留极性
- 预期：IRI改善达到70.9%峰值

## 常见陷阱

### 陷阱1：位置索引漂移
- **症状**：`error: 位置X期望Y但序列中是Z`
- **原因**：前几轮突变改变了序列，未更新位置映射
- **解决**：每轮突变后立即记录完整序列并逐位核对（建议用纸笔或序号标注）
- **实例**：R4的A19E实际将E放在20位，导致后续R6的A20Q变为Q20→T20

### 陷阱2：禁区触碰
- **症状**：`CRITICAL` 警告，`has_critical_warnings: true`
- **解决**：立即回退。A6-A12即使推荐Ala→Ser也不可触碰
- **实例**：A11S触发禁区警告

### 陷阱3：过早放弃
- **现象**：前5轮TH模拟始终0.1°C，评估TH缓慢上升
- **真相**：TH突破需要Thr密度越过~15个阈值
- **标准**：在第6轮评估TH应≥0.7°C，模拟TH应突破至1.0°C

### 陷阱4：C端密集区引入Glu
- **症状**：T29E导致IRI恶化（-2.6%）
- **规则**：Glu仅在非IBS面中部（如19位）使用，C端T28-T37区域保持纯Thr阵列

### 陷阱5：Glu→Asp缩短侧链
- **症状**：E19D无任何性能变化
- **原理**：Glu侧链长度恰好匹配冰面氧原子间距，缩短破坏匹配

## 验证方法

### 每轮检查清单
1. `forbidden_check.passed` = `true`
2. `ibs_integrity_check.passed` = `true`
3. `has_critical_warnings` = `false`
4. 突变后序列长度 = 37（不变）

### 阶段里程碑验证
- **R3后**: IRI改善≥40%，Thr≥9个
- **R5后**: IRI改善≥60%，Thr≥15个
- **R6后**: TH +233%，IRI改善≥64%（突破确认！）
- **R9后**: IRI改善≥69%，Thr≥20个
- **R15后**: IRI改善≥70%

### 最终验证
- `afp_design_summary` 确认所有指标
- geometry_score 始终 ≥0.9（0.933为最优）
- AFP概率 ≥99.9%
- 表达评分 ≥0.6（较原始提升≥75%）

## 设计规则总结

| 规则 | 证据强度 |
|------|----------|
| Thr密度>15触发TH突破 | 强（本次+技能历史） |
| C端密集Thr区禁止Glu | 中（T29E恶化IRI） |
| Glu在非IBS面中部效果最优 | 强（A19E +52% IRI） |
| 禁区即使推荐Ala→Ser也不可触碰 | 强（A11S触发CRITICAL） |
| Glu侧链长度不可缩短 | 中（E19D无效果） |
| 非IBS面Glu→Gln可微调IRI | 弱（E19Q +1.4% IRI） |
