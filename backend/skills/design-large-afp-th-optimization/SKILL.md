# 大型AFP TH活性优化设计技能

## 何时使用

- 序列长度 >100 aa 的AFP（Type III、昆虫超活性、细菌IBP等）
- Type I AFP技能（37aa骨架）不适用的大蛋白
- 基线TH很低（<0.2°C）需要翻倍提升
- 序列无明确IBS或IBS残基质量低（residue_quality < 0.6）
- 需要在不破坏整体折叠的前提下全局优化

## 核心设计原则

### 黄金法则：Thr密集化
- **Thr越多越好**：从基线23 Thr → 53 Thr (+130%) 是可行的
- **优先转化路径**：S→T > N→T > G→T > A→T > Q→T > L→T
- **每3个Thr约提升TH 0.01-0.02°C**（大型蛋白中）
- **关键阈值**：约40+ Thr时TH突破加速，50+ Thr时趋于饱和

### 禁区规避
- N端[1-12]区域绝对不可突变（afp_knowledge_query返回的forbidden_positions）
- 原始序列中的保守疏水核心残基（V18, L23等）保持不动
- 不要在IBS预测区域引入带电残基

### 电荷守恒
- **净电荷必须保持不变**（本例中保持-8.5）
- 引入Glu（E）导致净电荷偏移会严重损害AFP概率（-35%）
- 如意外引入E，立即用E→T修复

## 操作步骤

### 阶段一：基线建立（3步）

1. **afp_knowledge_query** — `query_intent="full_analysis"`
   - 记录：IBS位置、禁区列表、Thr含量、净电荷、基线TH/IRI/表达/稳定性/AFP概率
   - 注意：大型蛋白的IBS预测可能不准确（残基质量低），这是正常的

2. **afp_design_strategy** — 获取初始推荐位点
   - 传入基线性能数据
   - 记录推荐的mutation_hotspots（主要是Ala→Ser位点）

3. **afp_ice_bind_simulate** — 基线模拟
   - 使用 `target_ice_plane="auto"` 自动检测
   - 记录geometry_score和activity_assessment

### 阶段二：Thr密集化（核心5-8轮）

4. **第1轮：高密度Thr区域优先**
   - 在已有Thr簇附近引入A→T、G→T、S→T
   - 选择3个位点，间距<5 aa形成连续Thr链
   - 示例：60-80区域的A60T, G66T, A78T

5. **第2-3轮：Thr簇扩展**
   - 在另一个Thr富集区域创建三联Thr簇（如S144T, S145T形成T143-T144-T145）
   - L→T可用于桥接两个Thr（如L152T桥接T150-T153）
   - 目标：每轮+3 Thr

6. **第4-5轮：跨区域Thr分布**
   - 在稀疏区域（如C端、中间域）添加G→T、A→T
   - 桥接相距3-7 aa的Thr间隙
   - 避免所有Thr集中在单一区域

7. **第6-8轮：填充间隙**
   - N→T、Q→T转化（将非最优冰结合残基转为Thr）
   - 在C端构建第二冰结合面（T216-T219-T221-T225区域）
   - 目标：达到40+ Thr

### 阶段三：陷阱修复（如适用）

8. **Glu陷阱检测与修复**
   - ⚠️ **禁止主动引入Glu**（除非afp_design_strategy明确推荐且蛋白为Type I）
   - 如AFP概率突然暴跌>15%，检查是否有E被引入
   - 修复方法：E→T（在下一轮立即执行）
   - 修复效果：单轮可恢复+14% AFP概率

9. **净电荷监控**
   - 每轮后检查physchem_profile中的net_charge
   - 确保与基线偏差不超过±2
   - 如果偏差过大，逆转最近的带电残基突变

### 阶段四：表达优化（2-3轮）

10. **Ala→Ser溶解度提升**
    - 在非IBS区域选择A→S（知识库推荐的mutation_hotspots）
    - 每轮2-3个A→S突变
    - 预期表达提升：每3个A→S约+1-2%表达

11. **其他溶解度突变**
    - G→S（在非关键区域）
    - 优先在表面暴露的Ala/Gly位点

### 阶段五：收尾与总结

12. **最后1-2轮微调**
    - 如果TH仍有上升空间：N→T、Q→T
    - 如果TH已饱和：专注于表达/稳定性微调

13. **afp_design_summary** — 生成最终报告

14. **afp_visualize_trajectory** — 可视化全流程

## 每轮必须执行

- **afp_mutate_sequence**: 先突变，检查warnings
- **afp_evaluate_mutation**: 每轮评估（验证禁区合规）
- **afp_ice_bind_simulate**: 每2-3轮或关键轮次执行（对比原始基线）
- **afp_design_strategy**: 每5轮获取更新策略

## 常见陷阱

### 陷阱1：位置确认错误（高频！）
- **症状**: `error: 位置X期望Y但序列中是Z`
- **发生率**: 本任务91次调用中出现8次（~9%）
- **原因**: 前几轮突变改变了序列，未更新位置索引
- **预防**: 
  - 每轮突变后记录完整序列
  - 下次突变前，逐位核对目标位置的当前氨基酸
  - 使用afp_sequence_analyze重新获取position_guide
- **快速修复**: 从错误信息获取实际氨基酸，修正from_aa参数

### 陷阱2：Glu引入导致AFP概率崩溃
- **症状**: AFP概率单轮暴跌15-35%，净电荷增大
- **案例**: Q133E+Q230E导致AFP概率0.581→0.336（-42%）
- **机制**: 大型蛋白中Glu负电荷破坏整体静电平衡
- **解决**: 下一轮立即E→T修复（效果：+14% AFP概率恢复）
- **预防**: 大型AFP（>100aa）避免主动引入Glu；Glu策略仅适用于Type I（37aa）

### 陷阱3：几何评分永久锁定
- **症状**: geometry_score在全部15轮中保持0.762不变
- **原因**: IBS检测锁定在[2,13,24,35]，残基质量固定0.51
- **影响**: 不影响TH/IRI的逐步改善，但限制了评估精度
- **缓解**: 接受此限制，关注TH prediction和AFP probability趋势

### 陷阱4：表达量与Thr数量的权衡
- **症状**: 高Thr轮次（R10-R11）表达评分从0.507降至0.486
- **原因**: Thr增加略微降低疏水性，影响折叠效率
- **解决**: 穿插Ala→Ser轮次恢复表达（R12-R14回升至0.526）

### 陷阱5：IRI改善停滞
- **症状**: R7之后IRI不再改善（锁定在2.50 µM）
- **原因**: Thr密集化主要改善TH，IRI需要IBS平坦性优化
- **缓解**: IRI 56%累积改善已足够；如需进一步IRI优化，尝试非IBS面极性调整

### 陷阱6：类型误判
- **症状**: R15时类型从Type III变为Insect hyperactive
- **原因**: N端N19T/Q21T改变了局部基序特征
- **影响**: 通常无害，甚至是正面信号（昆虫AFP通常TH更高）
- **不需特殊处理**

## 验证方法

### 每轮验证
1. `afp_evaluate_mutation.forbidden_check.passed` 必须为 `true`
2. `afp_evaluate_mutation.ibs_integrity_check.passed` 必须为 `true`
3. `has_critical_warnings` 必须为 `false`
4. 净电荷与基线偏差不超过±2

### 阶段验证（每5轮）
1. TH prediction应逐步上升（目标：+50% → +100% → +133%）
2. IRI IC50应逐步下降（目标：-20%以上）
3. AFP概率不应低于基线（0.52），目标>0.70
4. 表达评分不应低于0.47

### 最终验证
1. TH ≥ 0.20°C（从0.09基线）
2. AFP概率 ≥ 0.70
3. IRI改善 ≥ 50%
4. 表达评分 ≥ 基线
5. 净电荷 = 基线净电荷
6. 序列长度 = 原始长度（261 aa）

## 快速参考

### 突变优先级矩阵（大型AFP）
```
最优引入Thr:  S→T > N→T > G→T > A→T
可接受:      Q→T, L→T (仅在桥接必要时)
避免:        V→T, I→T, F→T (疏水核心)
禁止:        带电残基(D/E/K/R/H)→T (仅限修复E→T)
表达优化:    A→S > G→S
```
